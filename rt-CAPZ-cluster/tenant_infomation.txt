Create Service Principal in Azure
az ad sp create-for-rbac --name "capz-sp" --role Owner --scopes /subscriptions/d06f8f89-f7d3-46b3-b7a8-649244fb54c6

A service principal (SP) is a security identity used by apps, scripts, or tools to access Azure resources.

TRY WITHOUT FEDERATED IDENTITY
Also it is important to create the federated identity credentials -- otherwise it will be impossible to deploy.
How to do it:
az ad app federated-credential create \
  --id 5796b248-d3e7-45de-b98c-880aa403b2cb \
  --parameters '{
    "name": "capz-federated-identity",
    "issuer": "https://kubernetes.default.svc.cluster.local",
    "subject": "system:serviceaccount:capi-system:capi-controller-manager",
    "audiences": ["api://AzureADTokenExchange"]
  }'



export AZURE_SUBSCRIPTION_ID=d06f8f89-f7d3-46b3-b7a8-649244fb54c6

# Create an Azure Service Principal and paste the output here
export AZURE_TENANT_ID=cb5b3c56-8331-4862-8e2c-369b8684fdc0
export AZURE_CLIENT_ID=5796b248-d3e7-45de-b98c-880aa403b2cb
export AZURE_CLIENT_ID_USER_ASSIGNED_IDENTITY=$AZURE_CLIENT_ID # for compatibility with CAPZ v1.16 templates
export AZURE_CLIENT_SECRET=k3n8Q~pUg7jBS-VdqDQ~QdXRwls7TVFlahHlxbN1

# Settings needed for AzureClusterIdentity used by the AzureCluster
export AZURE_CLUSTER_IDENTITY_SECRET_NAME="cluster-identity-secret"
export CLUSTER_IDENTITY_NAME="cluster-identity"
export AZURE_CLUSTER_IDENTITY_SECRET_NAMESPACE="default"

# Create a secret to include the password of the Service Principal identity created in Azure
# This secret will be referenced by the AzureClusterIdentity used by the AzureCluster
kubectl create secret generic "${AZURE_CLUSTER_IDENTITY_SECRET_NAME}" --from-literal=clientSecret="${AZURE_CLIENT_SECRET}" --namespace "${AZURE_CLUSTER_IDENTITY_SECRET_NAMESPACE}"

# Finally, initialize the management cluster
clusterctl init --infrastructure azure


Before running
clusterctl generate cluster <cluster-name> 

We have to define these:
# Name of the Azure datacenter location. Change this value to your desired location.
export AZURE_LOCATION="swedencentral"

# Select VM types.

export AZURE_NODE_MACHINE_TYPE="D4ds_v5"

export AZURE_RESOURCE_GROUP="<ResourceGroupName>"

  # Cluster templates authenticate with Workload Identity by default. Modify the AzureClusterIdentity for ServicePrincipal authentication.

yq -i "with(. | select(.kind == \"AzureClusterIdentity\"); .spec.type |= \"ServicePrincipal\" | .spec.clientSecret.name |= \"${AZURE_CLUSTER_IDENTITY_SECRET_NAME}\" | .spec.clientSecret.namespace |= \"${AZURE_CLUSTER_IDENTITY_SECRET_NAMESPACE}\")" capi-rt.yaml
